# DecodeLabs Internship — Project 2: Data Classification Using AI

Industrial Training Kit submission for DecodeLabs (Batch 2026).

## Overview

This project builds a complete supervised learning pipeline using
K-Nearest Neighbors (KNN) to classify e-commerce order data by `OrderStatus`.

**Dataset:** `Dataset for Data Analytics - Sheet1.csv` (1,200 e-commerce orders)
**Target:** `OrderStatus` — 5 classes (Cancelled, Delivered, Pending, Returned, Shipped)
**Algorithm:** K-Nearest Neighbors, tuned via the elbow method

## Project Files

| File | Purpose |
|---|---|
| `main.py` | Entry point — loads and inspects the raw dataset |
| `prepare_data.py` | Cleans data, encodes categorical columns, scales features, splits train/test |
| `find_best_K.py` | Tests K = 1–30, plots the error-rate elbow curve, saves the optimal K |
| `trainAndEvaluate.py` | Trains the final KNN model, generates the confusion matrix and classification report |
| `SummaryReport.py` | Compiles all results into `PROJECT2_FINAL_REPORT.txt` |

## Generated Outputs

| File | Description |
|---|---|
| `k_tuning_curve.png` | Error rate vs. K value, used to pick the best K |
| `confusion_matrix.png` | Heatmap of predicted vs. actual classes on the test set |
| `classification_report.txt` | Precision, recall, F1-score per class |
| `PROJECT2_FINAL_REPORT.txt` | Full written summary of the project |
| `prepared_data.pkl`, `best_k.pkl`, `final_model.pkl` | Intermediate artifacts passed between scripts (regenerated by running the pipeline — not meant to be hand-edited) |

## How to Run

```bash
pip install pandas scikit-learn matplotlib seaborn joblib
python main.py
python prepare_data.py
python find_best_K.py
python trainAndEvaluate.py
python SummaryReport.py
```

## Results

- **Best K:** 15 (selected via elbow method)
- **Final accuracy:** ~23%, close to the 20% random-guess baseline for 5 balanced classes
- **Key takeaway:** the available order-level features (product, price, quantity,
  payment method, referral source) show little statistical relationship with
  final order status. The pipeline itself — cleaning, scaling, splitting, tuning,
  evaluating — is implemented and validated correctly; the limitation is the
  predictive power of this particular feature set for this particular target.

See `PROJECT2_FINAL_REPORT.txt` for the full breakdown.

## Skills Demonstrated

Data cleaning, feature encoding and scaling, train/test methodology,
hyperparameter tuning, model evaluation beyond raw accuracy (confusion
matrix, precision/recall/F1), and critical interpretation of results
against a statistical baseline.
